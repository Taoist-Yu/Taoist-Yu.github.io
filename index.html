关于原码，反码与补码的一点个人理解

以下只是关于补码的一些个人理解

十进制与二进制

日常人们计数一般用的是十进制数，然而在计算机中只能存储二进制数据，所以需要把十进制数转换成二进制进而存储在计算机中。

如：十进制数43,转化成二进制就是 101011，再经过一些处理就能存储再计算机内存中了

无符号整数在内存中的存储

  1. 将十进制数转化成2进制数，如43转化成101011
  2. 在二进制数的左边补0，补至最高位。假设我们用一个8位的内存存储数据，43将转换成00101011

注：若将十进制数转换成二进制数后超出了最高位，则超出部分会被遗弃。

  如257会转换成100000001，遗弃超出部分后就变成00000001。

  遗弃最高位就相当于对模数取模（自行百度模数的定义


有符号整数在内存中的存储

  • 有两种方法可以用来表示有符号整数，原码和补码，现在普遍采用的是补码表示法

原码


保留最高位作为符号位，0代表正，1代表负，剩下所有位表示数值

反码


在介绍补码之前，先介绍一下反码，反码本身没什么意义，只是求补码的一种中间运算。

关于反码的定义其实是存在争议的，反码有两个定义：

第一种： 将原码的所有位全部取反


第二种，正数和零的反码就是其原码，负数的反码是原码保留最高位，剩下位全部取反


被普遍采用的定义是第二种，如果没特殊说明，接下来我们所说的反码都是指第二种

补码


现代计算机以补码存储数据，一般计算出整数的补码然后存储在内存中

补码的计算方法：

  1. 正数的补码等于其原码。
  2. 负数的补码就是它的反码+1。

关于Too many or too few lines的解决方案

如果你的提交出现WA的原因是Too many or too few lines，那么注意一下你是否出现以下情况：
1.调试输出未删除
2.是不是加了文件忘记删
3.C++的朋友不要用getline读入字符串，直接用cin

评测环境比较玄学，我开始就是直接用getline读入的字符串，本地和洛谷IDE上跑的结果都没问题，但是一提交到OJ上就十个点全WA，原因是Too many or too few lines。后来我把getline换成了cin，就直接AC了。

至于为什么会这样我也不知道，但是如果你出现了Too many or too few lines的情况可以看看你是不是用了getline进行读入。

[洛谷P1908]逆序对——分治，归并排序

本题采用分治的思想，进而想到归并排序

先将数组划分成左右两个子区间，然后可知：

逆序对的总数 = 左区间逆序对数目+右区间逆序对数目+同时位于左右区间的逆序对的数目（_即X在左区间，Y在右区间的逆序对X,Y _）

对于左右区间的逆序对数，我们可以通过递归的方法求出，现在的问题就在于，如何求出同时位于左右区间的逆序对数目。

/类似于归并排序的合并过程/
//左区间[l,m)右区间[m,r)
//p1,p2,p分别为左右区间和辅助数组上的游标
int p1=l,p2=m,p=l;
while(p1<m && p2<r){
if(a[p1] <= a[p2]){
s[p++] = a[p1++];
} else {
ans += (m-p1);
s[p++] = a[p2++];
}
}
//以下就和普通的归并排序合并过程一样
while(p1<m)s[p++] = a[p1++];
while(p2<r)s[p++] = a[p2++];

for(p=l;p<r;p++)a[p] = s[p];
