80X86汇编语言知识点总结

80X86六大部件

  1. 总线接口部件
  2. 执行部件
  3. 指令预取部件
  4. 指令译码部件
  5. 分段部件
  6. 分页部件

80X86的寄存器组（隶属于执行部件）

  1. 数据寄存器组

  • AX
    累加器

  • BX
    基址寄存器

  • CX
    计数寄存器

  • DX
    数据寄存器

  1. 指示器变址寄存器组

  • SI
    源变址寄存器

  • DI
    目的变址寄存器

  • BP
    堆栈基址寄存器

  • SP
    堆栈指示器

标志寄存器

  1. 条件标志位
  • 符号标志SF
  • 零标志ZF
  • 溢出标志OF
  • 进位标志CF
  • 辅助进位标志AF
  • 奇偶标志PF
  1. 控制标志位
  • 方向标志DF
  • 中断允许标志IF
  • 跟踪标志TF

寻址方式

  1. 寄存器寻址
    格式: R
    示例: AX
  2. 寄存器间接寻址
    格式: [R]
    示例: AX
  3. 变址寻址
    格式: [R*F+V],[R*F]+V,V[R*F]
    示例: [EBX*2+5]
  4. 基址加变址寻址
    格式: [BR+IR*F+V],V[BR][IR*F],V[BR+IR*F]
    示例: 8[BX][SI]
  5. 立即寻址
    格式: n
    示例: 12H
  6. 直接寻址
    格式: [n]
    示例: [1000h]

另注:

  1. 上述六种寻址方式可归为三大类，寄存器方式，存储器方式，立即方式。
  2. 一条指令的源操作数和目的操作数不能同时为存储器方式表示。

汇编指令

  1. 堆栈操作指令

  • PUSH OPS
    将立即数或寄存器，段寄存器，存储器中的一个字/双字压入堆栈中。

  • POP OPD
    将栈顶元素弹出送至某一寄存器，段寄存器（除CS外）或存储器中。

  • PUSHA
    将8个16位寄存器AX、CX、DX、BX、SP、BP、SI、DI顺序入栈保存。

  1. 标志寄存器操作指令
  • LEAF
    把标志寄存器低八位送入AH中
  1. 数据传送指令
  1. 一般数据传送指令
    1. 传送指令
      • MOV OPD, OPS

        将OPS的内容送至OPD

      • MOVSX OPD, OPS

        扩展OPS符号位至与OPD相同的数据类型后送至OPD

      • MOVZX OPD, OPS

        OPS高位补0，扩展成与OPD相同的数据类型后送至OPD

    2. 数据交换指令
      • XCHG OPD, OPS

        OPS与OPD内容交换

      • BSWAP OPD

        32位寄存器的第一个字节与第四个字节交换，第二个字节与第三个字节交换。

      • XADD OPD, OPS

        先交换OPD和OPS，再将OPD和OPS的和送至OPD。

    3. 查表转换指令
      • XLAT OPS 或XLAT

        将以(BX)或(EBX)位首地址,(AL)为偏移地址的字节存储单元中的数据送入AL。具体使用(BX)还是(EBX)取决于16位段还是32位段。

  2. 地址传送指令
    1. 传送偏移地址指令
      • LEA OPD, OPS

        按OPS提供的寻址方式计算偏移地址，将结果送入OPD中。

    2. 传送偏移地址及数据段首址指令
      • LDS OPD, OPS

        OPS送入OPD，OPS+2或OPS+4送入DS。
        其中OPD是通用寄存器，OPS是DD或DF存储器

    3. 传送偏移地址及附加数据段指令
      • LES OPD, OPS

        OPS送入OPD，OPS+2或OPS+4送入ES。
        其中OPD是通用寄存器，OPS是DD或DF存储器

  1. 算术运算指令
  1. 加运算指令
    • INC OPD
    • ADD OPD, OPS
  2. 减运算指令
    • DEC OPD
    • NEG OPD
    • SUB OPD, OPS
    • CMP OPD, OPS
  3. 乘运算指令
    • IMUL OPD, OPS
    • IMUL OPD, OPS, n
    • IMUL OPS
    • MUL OPS
  4. 符号扩展指令
    • CBW

      AL的符号扩展至AH中

    • CWD

      将AX符号扩展到DX中

    • CWDE

      将AX中符号扩展至EAX

    • CDQ

      将EAX中有符号数扩展为EDX、EAX

  5. 除运算指令
    • DIV POS

      字节除法(AX)/(OPS)->AL(商),AH(余数)
      字节除法(DX,AX)/(OPS)->AX(商),DX(余数)
      字节除法(AX)/(OPS)->EAX(商),EDX(余数)
      除数是0或者运算结果溢出，会产生溢出中断，除法指令不会影响标志寄存器。

    • IDIV OPS

      字节除法(AX)/(OPS)->AL(商),AH(余数)
      字节除法(DX,AX)/(OPS)->AX(商),DX(余数)
      字节除法(AX)/(OPS)->EAX(商),EDX(余数)
      所得商符号与数学定义相同，余数符号与被除数相同。

  1. 位操作指令
  1. 逻辑运算指令
    • NOT OPD

      按位取反

    • AND OPD, OPS

      按位与（逻辑乘）

    • OR OPD, OPS

      按位或（逻辑加）

    • XOR OPD, OPS

      按位异或（按位加）

    • TEST OPD, OPS

      做逻辑乘（按位与）运算，根据结果设置标志位SF,ZF,PF，不改变OPS, OPD的内容

    • BT OPD, OPS

      取OPD的第OPS位送至CF
      OPD必须位16/32位的寄存器或存储单元，8位会自动转化为16位。
      OPS只能是立即数或与OPD同类型的寄存器，不能是存储单元
      OPS应在-128~255之间，若OPS绝对值大于OPD的位数，取OPS除16或32的余数。

  2. 移位指令
    • SAL OPD, n
    • SAR OPD, n
    • SHL OPD, n
    • SHR OPD, n
    • ROL OPD, n
    • ROR OPD, n
    • RCL OPD, n
    • RCR OPD, n
    • SHLD OPD, OPS, n
    • SHRD OPD, OPS, n
  1. 串操作指令

    • STOS

Codeforces-976D Degree Set [思维]

2019-01-20
算法

原题链接

  codeforces 976D Degree Set

大致题意

  
给出一个长度为n的度数集合d（输入数据为一个递增数组d且无重复元素），要求你构造一个有d[n]+1个顶点的无向图，使得改无向图中的点的度数的集合与给出的集合相等

  
要求：

  1. 构造的无向图中包含d[n]+1个顶点
  2. 没有自环
  3. 没有重边
  4. 边数小于10^6
  5. 无向图的度数集与给出的度数集相等

题解

以下所说的图均为无向图

两个结论

  1. 对于一个顶点数为n的完全图，度数集合为{n-1}

  2. 对于一个顶点数为n个顶点的图，取k∈[1,n]，令[1,k]中所有顶点与图中所有顶点连边，则[1,k]中所有顶点度数为n-1，[k+1,n]中所有顶点度数为k。

其中结论1是结论2的特例（取k = n）

题目分析

  
我们现在d[n]+1个顶点，需要将其构造成一个度数集为{d[1],d[2]….d[n]的无向图}

  
考虑结论2，我们令[ 1，d[1] ]中所有顶点与图上所有顶点连边，则可得[ 1, d[1] ]中所有顶点度数为d[n]，剩下所有顶点度数为d[1]。我们将图中顶点划分为3个部分

  此时我们的度数集合为{d[1], d[n]}, 接下来只需要在第2部分中连边，使得度数集合中出现d[2],d[3]…d[n-1]即可。

  
问题转化为：构造一个顶点数为d[n-1]-d[1]+1的无向图，使其度数集为{d[2]-d[1]，d[3]-d[1]，….d[n-1]-d[1]},这是一个与原问题类似的规模更小的问题，可以重复上述操作。

核心代码

我们每次都将1~d[1]中顶点与图上所有顶点连边，然后忽略1~d[1]与d[n-1]+2~d[n]+1,在区间d[2]~d[n-1]+1上重复同样的操作。

关于原码，反码与补码的一点个人理解

2018-10-10
计算机基础

以下只是关于补码的一些个人理解

十进制与二进制

日常人们计数一般用的是十进制数，然而在计算机中只能存储二进制数据，所以需要把十进制数转换成二进制进而存储在计算机中。

如：十进制数43,转化成二进制就是 101011，再经过一些处理就能存储再计算机内存中了

无符号整数在内存中的存储

  1. 将十进制数转化成2进制数，如43转化成101011
  2. 在二进制数的左边补0，补至最高位。假设我们用一个8位的内存存储数据，43将转换成00101011

注：若将十进制数转换成二进制数后超出了最高位，则超出部分会被遗弃。

  如257会转换成100000001，遗弃超出部分后就变成00000001。

  遗弃最高位就相当于对模数取模（自行百度模数的定义


有符号整数在内存中的存储

  • 有两种方法可以用来表示有符号整数，原码和补码，现在普遍采用的是补码表示法

原码


保留最高位作为符号位，0代表正，1代表负，剩下所有位表示数值

反码


在介绍补码之前，先介绍一下反码，反码本身没什么意义，只是求补码的一种中间运算。

关于反码的定义其实是存在争议的，反码有两个定义：

第一种： 将原码的所有位全部取反


第二种，正数和零的反码就是其原码，负数的反码是原码保留最高位，剩下位全部取反


被普遍采用的定义是第二种，如果没特殊说明，接下来我们所说的反码都是指第二种

补码


现代计算机以补码存储数据，一般计算出整数的补码然后存储在内存中

补码的计算方法：

  1. 正数的补码等于其原码。
  2. 负数的补码就是它的反码+1。

算法

本题采用分治的思想，进而想到归并排序

先将数组划分成左右两个子区间，然后可知：

逆序对的总数 = 左区间逆序对数目+右区间逆序对数目+同时位于左右区间的逆序对的数目（_即X在左区间，Y在右区间的逆序对X,Y _）

对于左右区间的逆序对数，我们可以通过递归的方法求出，现在的问题就在于，如何求出同时位于左右区间的逆序对数目。

很显然，对于任意一个左区间元素X，和任意一个右区间元素Y，X一定在Y的左边，也就是说X和Y在各自区间中的位置并不影响有序对（X,Y）是否为逆序对。
